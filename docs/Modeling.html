Introduction:

After doing an exploratory data analysis on four variables in relation to diabetic outcomes, I found I was most interested to see how Sex, Income and Education performed as predictors of Diabetes diagnosis. These were the four variables that, at a glance, showed possibly some effect on amount of diabetes. I also find it interesting that these predictors are not comorbidities. Rather, they are biological (sex) or socio-economic (income and education) categories that people fall into. This means that without having extra medical measurements done, providers could be aware of how the risk differs between patients, and perhaps patients would consent to early screenings, etc. Let’s see how good these variables are at predicting a diabetic diagnosis!

Logistic Regression Models:

A logistic regression model is a model we use when our response variable is binary. It uses the predictor variables to estimate the probability of the event occuring (in this case the outcome of diabetes). This model works with our data because our response variable is binary in this case (diabetes, or no diabetes) and we’re using sex, income and education as predictors!

Recipes for LR models:

LR1_rec <- recipe(Diabetes_binary ~ Sex+ Income+ Education, data = train)%>%
  step_dummy(all_nominal_predictors())

LR2_rec  <- recipe(Diabetes_binary ~ Sex + Income, data = train) %>%
  step_dummy(all_nominal_predictors())

LR3_rec <- recipe(Diabetes_binary ~ Income + Education, data = train) %>%

Classification Tree:

A classification tree is like a decision workflow that starts at a root predictor and where each node depending on the value of that predictor, splits into two sub-nodes. Starting with certain given values of predictors (sex, income, education) and following the tree you end up with a predicted outcome (in this case diabetes). Sometimes we try to use classification trees because they can easy to interpret and can pick up on interactions between the predictors! We will see if this is a better model than our LR model 1 with sex, income and education as predictors!

#tree recipe 
tree_rec <- recipe(Diabetes_binary ~ Sex + Income + Education, data = train) %>%
  step_dummy(all_nominal_predictors())

#model spec
tree_mod <- decision_tree(
  cost_complexity = tune(), 
  min_n = 20) %>%
  set_engine("rpart") %>%
  set_mode("classification")

#tree workflow
tree_wkf <- workflow() %>%
  add_recipe(tree_rec) %>%

Random Forest

A random forest uses the same concept as a classification tree except it’s a method where many classification trees are made by using random subsets of the data and the predictors, then it aggregates the predictions. Random forest will usually have better accuracy because of the repetition, whereas a single classification tree is more likely to be less accurate and over-fit. Here we will compare our random forest results to that of the LR model 1 to see if it’s better at predicting the outcomes of diabetes, using sex, income and education as predictors!
